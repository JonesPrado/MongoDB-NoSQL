🔴 Bloco 1 — Fundação
use <db>
Entra ou cria um banco
use dbloja
show dbs
Lista todos os bancos
cls / exit
Limpa o terminal / sai do mongosh
db.dropDatabase()
Deleta o banco atual
⚠ sem volta
insertOne({})
Insere um documento
db.col.insertOne({nome: "x"})
insertMany([])
Insere vários de uma vez
db.col.insertMany([{}, {}])
find(filtro, proj)
Busca documentos. 2º arg = projeção (1 mostrar, 0 esconder). _id vem sempre, esconda com _id:0
db.col.find({}, {nome:1, _id:0})
findOne(filtro)
Retorna só o primeiro resultado
updateOne(f, op)
Atualiza o primeiro que bater o filtro
db.col.updateOne({nome:"x"}, {$set:{preco:9}})
updateMany(f, op)
Atualiza todos que baterem o filtro
deleteOne(filtro)
Deleta o primeiro que bater o filtro
deleteMany(filtro)
Deleta todos que baterem. {} = apaga tudo
⚠ deleteMany({}) apaga tudo
🟠 Bloco 2 — Operadores de query
$gt / $gte
Maior que / maior ou igual
{preco: {$gt: 100}}
$lt / $lte
Menor que / menor ou igual
{estoque: {$lte: 15}}
$ne
Diferente de (um valor só). Use $nin pra lista
{marca: {$ne: "Nike"}}
$in
Valor dentro de uma lista (OU)
{marca: {$in: ["Nike","Puma"]}}
$nin
Valor fora de uma lista
{marca: {$nin: ["Adidas"]}}
$and
Todos os filtros verdadeiros. Necessário quando dois filtros atingem o mesmo campo
{$and: [{marca:"Nike"},{preco:{$gt:100}}]}
$or
Pelo menos um filtro verdadeiro
{$or: [{marca:"Nike"},{preco:{$lt:50}}]}
$not
Inverte o filtro
{preco: {$not: {$gt: 100}}}
$exists
Verifica se campo existe (true) ou não (false)
{nome: {$exists: false}}
$type
Filtra pelo tipo do campo
{preco: {$type: "number"}}
$regex
Busca por padrão de texto. ^ = começa, $ = termina
{nome: {$regex: "^J"}}
$expr
Compara dois campos do mesmo documento entre si
{$expr: {$lt: ["$estoque","$preco"]}}
🟡 Bloco 3 — Operadores de update
$set
Altera ou cria um campo
{$set: {preco: 99}}
$unset
Remove um campo. Valor sempre ""
{$unset: {estoque: ""}}
⚠ valor sempre "" independente do tipo
$inc
Incrementa (negativo = decrementa)
{$inc: {estoque: -5}}
$mul
Multiplica o valor do campo
{$mul: {preco: 1.1}}
$rename
Renomeia um campo
{$rename: {preco: "valor"}}
⚠ novo nome sempre entre aspas
$min
Atualiza só se novo valor for MENOR que o atual
{$min: {preco: 199}}
$max
Atualiza só se novo valor for MAIOR que o atual
{$max: {estoque: 200}}
$setOnInsert
Só executa ao CRIAR via upsert. Ignorado em updates normais
{$setOnInsert: {marca:"X"}}, {upsert:true}
🟢 Bloco 4 — Arrays
$push
Adiciona item no final do array. Aceita duplicatas
{$push: {compras: "Tênis"}}
$addToSet
Igual ao $push mas ignora duplicatas
{$addToSet: {compras: "Meia"}}
$pull
Remove todos os itens que batem o critério
{$pull: {compras: "Meia"}}
$pullAll
Remove lista exata de valores. Sem condição
{$pullAll: {compras: ["Meia","Boné"]}}
$pop
Remove o primeiro (-1) ou último (1) item
{$pop: {compras: -1}}
⚠ só aceita 1 ou -1
$each
Usado com $push ou $addToSet pra inserir vários itens
{$push: {compras: {$each: ["A","B"]}}}
⚠ sem $each, array vira item único
$position
Insere em posição específica. Usado com $each
{$push: {compras: {$each:["X"], $position:0}}}
$slice
Limita tamanho do array após push. Usado com $each
{$push: {compras: {$each:["X"], $slice:3}}}
$sort (array)
Ordena o array após inserir. Usado com $each. 1=A→Z, -1=Z→A
{$push: {compras: {$each:["X"], $sort:1}}}
$all
Array precisa ter TODOS os valores (E). Diferente de $in (OU)
{compras: {$all: ["Meia","Jaqueta"]}}
$elemMatch
Condições devem bater no MESMO objeto do array. Evita falso positivo
{pedidos: {$elemMatch: {item:"Tênis", valor:{$gt:200}}}}
$size
Busca arrays com tamanho exato. Não aceita $gt/$lt
{compras: {$size: 3}}
⚠ só número exato, sem operadores
🔵 Bloco 5 — Avançado
count()
Conta quantos documentos batem o filtro
db.col.find({marca:"Nike"}).count()
sort()
Ordena o resultado. 1=crescente, -1=decrescente
db.col.find().sort({preco: -1})
pretty()
Formata o resultado legível (automático no mongosh moderno)
db.col.find().pretty()
findOneAndDelete()
Acha, retorna e deleta em uma tacada
db.col.findOneAndDelete({nome:"X"})
findOneAndUpdate()
Acha e atualiza. Retorna antes por padrão. Use returnDocument:"after"
db.col.findOneAndUpdate({nome:"X"}, {$set:{p:1}}, {returnDocument:"after"})
findOneAndReplace()
Substitui o documento inteiro. Não usa $set
db.col.findOneAndReplace({nome:"X"}, {nome:"X", preco:9})
⚠ não use $set — passa o doc novo direto
new Date() / ISODate()
Armazena data. Sem aspas = tipo Date, comparável com $gt/$lt
{criadoEm: new Date("2024-01-15")}
⚠ string "2024-01-15" não é tipo Date
typeof
JS puro no terminal. Verifica tipo de um valor
typeof "texto" // "string"
